# Awesome DESIGN.md

将 DESIGN.md 复制到你的项目中，告诉你的 AI 智能体"帮我构建一个看起来像这样的页面"，即可获得真正还原设计的像素级完美 UI。


## 什么是 DESIGN.md？

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是 Google Stitch 引入的一个全新概念——一份纯文本设计系统文档，AI 智能体可以读取它来生成一致的 UI。

它就是一个 markdown 文件。不需要 Figma 导出，不需要 JSON schema，不需要任何特殊工具。将其放入项目根目录，任何 AI 编程智能体或 Google Stitch 都能立刻理解你的 UI 应该是什么样子。Markdown 是 LLM 最容易读取的格式，因此无需解析或配置。

| 文件 | 谁读取它 | 它定义什么 |
|------|---------|-----------|
| `AGENTS.md` | 编程智能体 | 如何构建项目 |
| `DESIGN.md` | 设计智能体 | 项目的外观和风格 |

**本仓库提供开箱即用的 DESIGN.md 文件**，从真实网站中提取。

## 请求 DESIGN.md

你可以为特定网站[请求 DESIGN.md](https://getdesign.md/request)，包含仅向你交付的私密请求。

## 每个 DESIGN.md 包含什么

每个文件遵循 [Stitch DESIGN.md 格式](https://stitch.withgoogle.com/docs/design-md/format/)，并附带扩展章节：

| # | 章节 | 涵盖内容 |
|---|------|---------|
| 1 | 视觉主题与氛围 | 情绪、密度、设计理念 |
| 2 | 色彩调色板与角色 | 语义名称 + 十六进制色值 + 功能性角色 |
| 3 | 排版规则 | 字体家族、完整层级表 |
| 4 | 组件样式 | 按钮、卡片、输入框、导航及其状态 |
| 5 | 布局原则 | 间距刻度、网格、留白理念 |
| 6 | 深度与层级 | 阴影系统、表面层级 |
| 7 | 规范与禁忌 | 设计护栏和反模式 |
| 8 | 响应式行为 | 断点、触控目标、折叠策略 |
| 9 | AI 智能体提示指南 | 快速颜色参考、开箱即用的提示词 |

每个站点包含：

| 文件 | 用途 |
|------|------|
| `DESIGN.md` | 设计系统（AI 智能体读取的内容） |
| `preview.html` | 可视化目录，展示色样、字号层级、按钮、卡片 |
| `preview-dark.html` | 同上目录，暗色表面版本 |

### 如何使用


1. 将某个站点的 `DESIGN.md` 复制到你的项目根目录
2. 告诉你的 AI 智能体使用它。
